# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Repository Purpose

This is **SET React + Shadcn Template** - a production-ready starter template for teams. This is NOT an application project. Teams will clone this to start new projects, so the structure and conventions are critical and must be preserved.

**Critical Rule**: The folder structure is SACRED. Teams will replace content but NOT the structure. When modifying this template, preserve the organizational pattern.

## Architecture Overview

### Folder Structure (MUST NOT CHANGE)

```
src/
├── shadcn/                    # AUTO-GENERATED by shadcn CLI - NEVER edit manually
│   ├── components/            # All shadcn UI primitives
│   └── lib/utils.ts           # cn() utility
├── components/                # Custom shared components
├── navigation/                # Navigation components (sidebar, header, nav items)
├── pages/                     # Route components (page per file)
├── apis/                      # API calls and backend integration
├── hooks/                     # Custom React hooks
├── App.tsx                    # Routes configuration + route metadata
├── main.tsx                   # Entry point
└── index.css                  # Theme CSS variables (TweakCN compatible)
```

**Critical separations:**
- `shadcn/` = Generated code (shadcn CLI manages this)
- `navigation/` = Navigation UI and logic
- `pages/` = Route components only
- `apis/` = Backend integration
- `components/` = Reusable custom components

### Configuration Files

**components.json** - Shadcn CLI configuration:
- Points to `src/shadcn/components` for component generation
- Aliases configured for `@/shadcn/components`, `@/shadcn/lib/utils`
- DO NOT modify aliases - they're connected to tsconfig paths

**vite.config.ts** - Build configuration:
- `@/` alias → `./src/`
- Tailwind CSS v4 via `@tailwindcss/vite` plugin

**tsconfig.json**:
- Path alias: `@/*` → `./src/*`
- `verbatimModuleSyntax: false` (required for Vite compatibility)
- `jsx: "react-jsx"` (React 19)

## Routing Architecture

The template uses React Router v7 with a centralized metadata system:

**src/App.tsx**:
```typescript
export const routes = [
  { path: "/", title: "Home" },
  { path: "/dashboard", title: "Dashboard" },
  // ... route metadata
]
```

This `routes` array is the **single source of truth** for:
1. Route definitions in `<Routes>`
2. Page titles (consumed by `src/navigation/site-header.tsx`)
3. Documentation of available routes

**Adding a new route requires 3 steps:**
1. Create page component in `src/pages/`
2. Add route to `routes` array with `{ path, title }`
3. Add `<Route>` element in the JSX

The header dynamically displays the title by looking up `location.pathname` in the `routes` array.

## Navigation System

Navigation is split into modular components in `src/navigation/`:

- **app-sidebar.tsx** - Main sidebar, contains navigation data structure
- **site-header.tsx** - Top header with dynamic title (reads from `routes` in App.tsx)
- **nav-main.tsx** - Main navigation items (supports collapsible sub-items)
- **nav-projects.tsx** - Projects section
- **nav-user.tsx** - User profile dropdown
- **team-switcher.tsx** - Organization/team switcher

**Navigation data flow:**
1. `app-sidebar.tsx` defines the nav structure with `navMain` and `projects` arrays
2. Components use React Router's `useLocation()` to detect active routes
3. `isActive` prop highlights current page

## Theme System & Styling Rules

**CRITICAL: `src/index.css` is SACRED - NEVER modify it.**

Themes use OKLCH color space via TweakCN (https://tweakcn.com/editor/theme).

**src/index.css is ONLY modified by:**
- Users manually (when changing themes from TweakCN)
- Shadcn CLI (when adding components that need CSS variables)
- **NEVER by Claude Code**

**Styling Approach:**

1. **Use Tailwind CSS v4 utility classes ONLY when explicitly requested**
   - Only use Tailwind if user asks for styling changes
   - Otherwise, leave styling alone

2. **Allowed Tailwind classes (when needed):**
   - **Spacing**: `p-4`, `m-2`, `gap-4`, `space-x-2`, etc.
   - **Positioning**: `flex`, `grid`, `absolute`, `relative`, etc.
   - **Layout**: `flex-col`, `items-center`, `justify-between`, etc.

3. **DO NOT use Tailwind classes for:**
   - Colors (use theme variables automatically via shadcn components)
   - Typography (fonts, sizes - theme handles this)
   - Borders, shadows, radius (theme handles this)
   - Background colors, text colors (theme handles this)

4. **DO NOT create:**
   - Additional CSS files
   - CSS modules
   - Styled-components or CSS-in-JS
   - Inline styles (except in rare cases)

**src/index.css structure (DO NOT TOUCH):**
```css
/* FRAMEWORK IMPORTS - DO NOT MODIFY */
@import "tailwindcss";

:root {
  /* Theme variables - users replace from TweakCN */
  --background: oklch(...);
  /* ... all CSS variables */
}

.dark {
  /* Dark theme variables */
}

@theme inline {
  /* Tailwind mappings - DO NOT MODIFY */
}

@layer base {
  /* Base styles - DO NOT MODIFY */
}
```

**When users want to change themes (they do this, not you):**
- Replace the ENTIRE `:root` block from TweakCN
- Replace the ENTIRE `.dark` block from TweakCN
- DO NOT touch `@theme inline` or `@layer base` sections

**Dark mode:**
- Simple toggle in `src/components/mode-toggle.tsx`
- Uses class-based switching (adds `.dark` to `<html>`)
- No external theme libraries (intentionally simple)

## Import Conventions

Always use path aliases:
```typescript
// Shadcn components
import { Button } from "@/shadcn/components/button"

// Shadcn utilities
import { cn } from "@/shadcn/lib/utils"

// Custom components
import { Layout } from "@/components/layout"

// Navigation components
import { AppSidebar } from "@/navigation/app-sidebar"

// Pages
import Home from "@/pages/Home"

// Hooks
import { useIsMobile } from "@/hooks/use-mobile"
```

**Never use relative imports** (`../../`). Always use `@/` prefix.

## Common Template Modifications

When teams customize this template:

**1. Change theme:**
- Go to https://tweakcn.com/editor/theme
- Copy generated CSS
- Replace `:root` and `.dark` blocks in `src/index.css`

**2. Add new page:**
- Create component in `src/pages/`
- Add to `routes` array in `src/App.tsx`
- Add `<Route>` in JSX
- (Optional) Add nav item in `src/navigation/app-sidebar.tsx`

**3. Modify navigation:**
- Edit `data.navMain` in `src/navigation/app-sidebar.tsx`
- Structure: `{ title, url, icon, items? }`
- Supports nested items (collapsible)

**4. Add API integration:**
- Create functions in `src/apis/`
- Import and use in page components

**5. Update branding:**
- Change `name` in `package.json`
- Update `data.teams` in `src/navigation/app-sidebar.tsx`
- Update `data.user` in `src/navigation/app-sidebar.tsx`

## TypeScript Notes

**useIsMobile hook:**
- Must return `boolean` (not `boolean | undefined`)
- Initial state must be `false` (not `undefined`)
- This prevents NaN errors in components using the hook

**Import types:**
- Use `import type` for type-only imports when needed
- React Router types from `react-router-dom`
